# AI 看视频方法对比

本文档用于回答一个产品和技术问题：**我们的音频优先短 clip 方法，和现在常用的 AI 看视频方法相比，具体强在哪里、弱在哪里、能用哪些指标说明。**

结论先说：我们的方案不是替代所有视频理解方法，而是更适合 **长视频问答、需要时间点证据、希望少看帧但看得有目的** 的场景。它的核心优势是采样效率和问答可回溯性。

## 常见方法

| 方法 | 做法 | 优点 | 主要问题 |
|---|---|---|---|
| 均匀抽帧 | 每隔 N 秒抽一帧，送图像模型总结 | 简单、便宜、易实现 | 容易错过关键瞬间；不懂音频语义；对动作过程弱 |
| 场景切分/镜头边界 | 根据画面变化或 shot boundary 抽关键帧 | 比均匀抽帧更关注视觉变化 | 对“说话提到但画面变化不大”的信息不敏感 |
| 全帧/高密度采样 | 大量抽帧或低 fps 扫全片 | 覆盖率高 | 成本高、延迟高、冗余多，长视频不友好 |
| 整段视频直塞多模态模型 | 把视频文件或压缩视频直接给支持视频的模型 | 接口简单，主题粗扫快 | 长视频细节记忆不稳，时间线和指令跟随容易漂移 |
| ASR 文本总结 | 只转写音频，再让 LLM 总结 | 很快，适合讲座/口播 | 看不到画面，无法验证视觉证据 |
| RAG 视频索引 | 抽帧/字幕/对象检测后建向量库 | 适合搜索和企业知识库 | 预处理重，是否命中取决于索引粒度 |
| 流式模型/直播模型 | 持续消费短窗口视频流 | 适合监控、直播、实时告警 | 通常需要持续计算；长视频离线问答仍要记忆/索引 |
| 我们的方法 | ASR 建音频世界模型，再按问题/音频选 2-4 秒 clip 验证 | 看得少但有目的；时间点可回溯；适合长视频问答 | 依赖转写质量；不适合完全无声且视觉事件密集的视频 |

## 指标定义

下面这些指标比“看起来回答得不错”更适合做方法对比。

| 指标 | 含义 | 为什么重要 |
|---|---|---|
| Temporal Recall | 抽样是否命中关键事件时间点 | 长视频里，没抽到关键时刻就无法回答 |
| Evidence Hit Score | 模型回答是否命中预设证据关键词组 | 衡量识别准确率和信息完整度 |
| Pass@0.67 | 命中分达到 2/3 以上的比例 | 粗略表示这个片段是否足以支撑问答 |
| Visual Budget | 实际送给视觉模型看的视频秒数/帧数 | 衡量成本和可扩展性 |
| Latency | 单次视觉调用耗时 | 影响用户等待和直播/准实时场景 |
| Payload Size | 每次请求传输的媒体大小 | 影响本地/网络模型吞吐 |
| Traceability | 回答能否回到具体时间点 | 影响用户信任和可验证性 |
| Robustness | 模型是否容易被无关上下文带偏 | 长视频和多主题视频尤其重要 |

## 本项目实测

实测样本：

- `3d_house`：17 分 34 秒，3D 打印装修长视频。
- `pocket4p`：12 分 41 秒，Pocket 4P 产品评测视频。
- 总时长：1815.21 秒。
- 关键 probe 点：8 个，覆盖施工现场、3D 打印家具、家具设计、稳定性、长焦、动态范围、发热等。

每个 probe 点都有预设关键词组，例如：

- `3D 打印家具 / 分层纹理 / 成本`
- `ISO1600 / 动态范围 / 低光测试`
- `长焦 / 三倍视角 / 60mm`

### 核心结果

| 方法 | 视觉预算 | 平均视觉调用耗时 | 平均命中分 | Pass@0.67 | 时间命中 | 平均载荷 | 结论 |
|---|---:|---:|---:|---:|---:|---:|---|
| 均匀稀疏抽帧 | 12 帧/视频 | 未跑模型 | - | - | 1/8 = 12.5% | - | 同样稀疏预算下，大多数关键点根本抽不到 |
| 音频选点 + 单帧 | 8 帧 | 1.415s | 0.833 | 62.5% | 8/8 | 35.7KB | 很快，但动作/切换/上下文不足 |
| 音频优先 2s clip | 看 16s，占总时长 0.88% | 2.152s | 1.000 | 100% | 8/8 | 46.0KB | 本轮最佳速度/准确率平衡 |
| 音频优先 4s clip | 看 32s，占总时长 1.76% | 2.164s | 1.000 | 100% | 8/8 | 81.8KB | 当前默认，更适合动作和上下文 |
| 8s clip | 看 64s，占总时长 3.53% | 2.135s | 0.875 | 75% | 8/8 | 144.8KB | 上下文更多，但会引入噪声 |
| 整段 320p/1fps 粗扫 | 全片低帧率 | 2.195s + 编码 3.145s | 0.666 | - | 全片粗扫 | 约 6800KB | 主题粗扫快，但细节记忆不稳 |

### 采样效率

这是最能说明差异的指标。

- 如果每条视频均匀抽 12 帧，只能在 +/-4 秒内命中 1/8 个关键 probe 点，时间命中率 12.5%。
- 如果希望用均匀抽帧保证 +/-4 秒覆盖，两条视频合计约需要 228 帧。
- 如果希望保证 +/-2 秒覆盖，两条视频合计约需要 455 帧。
- 我们的方法只看 8 个音频引导点，每个 4 秒，总共看 32 秒，占 1815 秒总时长的 1.76%，但 probe 点全命中。

这说明我们的核心价值不是“视觉模型本身更强”，而是 **把视觉模型调用放在更该看的时间点上**。

## 与常用方案逐项比较

### 1. 对比均匀抽帧

均匀抽帧最大问题是时间命中率。长视频里的关键信息往往只占几秒，如果预算只有 8-24 帧，很容易错过。

本轮实测中，均匀 12 帧/视频只命中 12.5% 的关键点。我们的音频引导采样命中 8/8。

结论：

- 均匀抽帧适合快速封面级概览。
- 音频优先采样更适合问答、证据和长视频摘要。

### 2. 对比单帧关键帧

音频选点后的单帧已经比均匀抽帧好很多，因为时间点选对了。但单帧仍然缺少动作和上下文。

本轮实测：

- 单帧平均命中分：0.833。
- 2s/4s clip 平均命中分：1.000。
- 单帧 Pass@0.67：62.5%。
- 2s/4s clip Pass@0.67：100%。

典型提升：

- `长焦 / 三倍视角`：单帧只看到设备，clip 能看到字幕和镜头上下文。
- `动态范围 / ISO1600`：单帧只看到暗场，clip 能看到暗部测试和图表切换。
- `稳定性样片`：clip 能确认骑马/户外运动样片，而不是只看一张图。

结论：

- 单帧适合静态物体、文字、场景判断。
- clip 更适合动作、样片、切换、过程、对比和产品评测。

### 3. 对比整段视频直塞模型

整段视频直塞最大的优点是接口简单，而且粗扫主题很快。问题是长视频细节不稳。

本轮整段低码率粗扫：

- 3D 打印视频：能识别主题，score=1.0。
- Pocket 4P 评测：跑偏成 Pocket 4 与 AirPods 对比，score=0.333。
- 平均 score=0.666。

结论：

- 整段视频直塞适合作为 coarse scan 或兜底主题识别。
- 不适合作为长视频问答的唯一记忆来源。
- 它可以和我们的方法结合：先整段粗扫判主题，再由音频/问题驱动短 clip 精查。

### 4. 对比只看音频/字幕

ASR + LLM 非常适合讲座、访谈、播客、口播视频；但遇到产品评测、施工、比赛、直播画面，视觉证据很关键。

我们的方案保留 ASR 的优势，但把它变成视觉注意力分配器：

- 音频负责决定“哪里值得看”。
- 短 clip 负责验证“画面到底有没有”。
- 最终回答保留时间点，用户能跳回视频核对。

结论：

- 纯音频适合内容主线。
- 音频 + clip 适合可验证视频问答。

### 5. 对比高密度抽帧 / 视频 RAG

高密度抽帧和 RAG 适合企业知识库和离线索引，但预处理更重。

按本轮两条视频估算：

- 为了 +/-2 秒时间覆盖，均匀采样约需要 455 帧。
- 我们实际只验证 8 个时间点，4 秒 clip 总视觉预算 32 秒。

结论：

- 如果目标是全库检索，RAG 更系统。
- 如果目标是对单个上传视频快速问答，音频优先短 clip 成本更低。

## 我们方法的优势

1. **采样效率高**：不追求看完整视频，而是看音频和问题提示的关键位置。
2. **信息密度高**：2-4 秒 clip 比单帧多出动作和前后关系，但又不会像 8 秒或整段那样引入太多噪声。
3. **可回溯**：每个结论能落回时间点，适合前端点击跳转。
4. **长视频友好**：视频越长，均匀抽帧越浪费；音频引导的优势越明显。
5. **可组合**：可以接 JoyAI、OpenAI-compatible vision、本地 ASR、web search、直播合规监控。

## 我们方法的不足

1. **依赖音频质量**：如果没有音频、转写错、或者关键视觉事件完全没人提到，音频引导会漏点。
2. **不适合纯视觉密集视频**：例如无解说的体育集锦、监控异常、游戏操作高频变化，需要额外视觉候选扫描。
3. **需要抽 clip 的工程复杂度**：比单帧 API 多 FFmpeg、缓存和失败回退逻辑。
4. **不是论文级 benchmark**：当前数据集只有两条代表视频、8 个 probe 点，需要扩展到更多类型。

## 推荐产品策略

| 场景 | 推荐方法 |
|---|---|
| 1-30 分钟上传视频问答 | 音频优先 + 2/4 秒短 clip |
| 产品评测、施工、教程 | 音频优先 + 4 秒 clip，必要时补看 |
| 纯口播/讲座 | ASR + LLM，少量封面/章节帧 |
| 无音频视频 | 视觉候选扫描 + 均匀/场景抽帧 |
| 直播合规 | 短窗口音频规则 + 单帧视觉审核，只保留违规证据 |
| 粗略判断视频主题 | 整段 320p/1fps 粗扫可作为辅助 |
| 企业级视频库搜索 | 建立字幕/帧/对象 RAG 索引 |

## 如何复现实验

运行多维 benchmark：

```powershell
python scripts\benchmark_multidim.py
```

生成方法对比摘要：

```powershell
python scripts\compare_methods_from_benchmark.py data\benchmarks\20260626_162946\report.json
```

冒烟测试：

```powershell
python scripts\benchmark_multidim.py --limit-points 1 --skip-whole
```

输出目录：

```text
data/benchmarks/{run_id}/report.json
data/benchmarks/{run_id}/report.md
```

## 下一步指标扩展

要让对比更有说服力，建议继续补这些维度：

- 视频类型：口播、教程、产品评测、体育、游戏、监控、直播。
- 数据量：至少 30-50 条视频，每条 5-20 个标注 probe 点。
- 指标：Temporal Recall、Evidence Hit Score、QA exactness、hallucination rate、latency、cost、payload、human preference。
- Baseline：均匀抽帧、scene detection 抽帧、整段视频模型、ASR-only、密集抽帧 RAG。
- 人工评分：回答是否有章法、是否答到问题、是否有时间点、是否能回放验证。
